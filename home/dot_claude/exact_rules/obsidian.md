# Obsidian as Default Writing Tool

When asked to create, write, or produce any document, plan, research summary, note, or written artifact, use the Obsidian skill (`/obsidian`) to create it in the Obsidian vault — not as a local file in the working directory.

## Applies To

- Research summaries and findings
- Design plans and architecture documents
- Meeting notes and agendas
- Project plans and task breakdowns
- Knowledge base articles
- Any markdown document not tied to a specific codebase

## Exceptions

- Code files, configs, and scripts belong in their project directory
- README, CHANGELOG, and repo documentation belong in the repository
- Files the user explicitly asks to create at a specific path
