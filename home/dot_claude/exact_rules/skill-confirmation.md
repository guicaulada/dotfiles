# Skill Confirmation

Before invoking any of the following skills, always ask the user for confirmation first. State which skill you intend to use and why, then wait for approval before proceeding.

- **create-pr** — Use when creating pull requests
- **issue** — Use when creating GitHub issues
- **review-deps** — Use when reviewing dependency bump PRs
- **review-pr** — Use when reviewing pull requests

These skills perform external actions (GitHub API calls, browser opens) that the user should explicitly approve.
